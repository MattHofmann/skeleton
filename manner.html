コンセプト

メンテナンス性、効率を意識したコーディング

  1. 再利用可能な形でつくる
  2. 同じ構造の同じスタイルのものは全ファイルで同じ構造の HTML CSS を利用
  3. 柔軟性を意識する
  4. 要素の追加・削除のし易さを意識する

文書型等

デフォルトの文書型は HTML5

デフォルトの文字コードは UTF-8

ターゲットブラウザに IE8 など HTML5 未対応のブラウザが含まれる場合には新要素をスタイルのために使用しない
※アウトラインなど純粋にセマンティクスを行うためには使用可

ID Class の名前の付け方

  • コンテンツに依存した名前は使用せず（例：× .company-title）、文書構造や UI 上の意味的な名前を付ける（例：○ .title #nav）
  • 同様に特定の色や形に依存した名前も、色や形が変更になった場合に名前と整合性がなくなるので極力避ける
  • 同じ意味の要素がページ毎に異なる場合は共通のスタイルは継承し、クラスを追加して必要な部分のみ上書きする（例：..title.title-**** のように継承元のクラス名前を先にし、ハイフンつなぎで文字列を追加）
  • 意味の判断のつかないものに関しては一般的な名前や意味の広い名前を付ける（例：.box .heading-a）

    上記の理由からクラス名の中で

    • .section
    • .unit
    • .box
    • .inner
    • .item
    • .heading
    • .more

    は汎用的なクラスとして使用するため、直接スタイルを指定しない。

    × .box { color: #000; } ○ .unit-2col .box { color: #000; }

JavaScript で振る舞いを指定する ID Class には極力スタイルを付けない

装飾が異なるが振る舞いが同じことがあるため、装飾で使用する名前と振る舞いで利用する名前は極力分離する。

プロパティがひとつのものは一行で。二行以上のものは改行。

メディアクエリ

メディアクエリは後で指定が変更しても柔軟に対応できるよう CSS ファイルにインラインで指定

× <link rel="stylesheet" media="screen and (min-device-width:768px" ○ @media only screen and (min-device-width:768px){}
